# 🛒 WB Price Tracker Bot

Телеграм-бот для автоматического отслеживания цен на товары с маркетплейса **Wildberries.by**. Бот уведомляет пользователя, если цена на выбранный товар снизилась или выросла.

## ✨ Основные возможности
- ➕ **Добавление товаров**: Просто отправьте боту ссылку на товар с Wildberries.
- 📉 **Автоматический мониторинг**: Бот проверяет актуальные цены каждый час.
- 🔔 **Умные уведомления**: Сообщение при изменении цены с указанием разницы в BYN и процентах.
- 📋 **Мой список**: Удобный просмотр всех отслеживаемых позиций.
- ❌ **Удаление**: Быстрое удаление товара из списка через Inline-кнопки под сообщениями.

## 🛠 Стек технологий
- **Язык**: Python 3.10+
- **Библиотека**: [pyTelegramBotAPI](https://github.com/eternnoir/pyTelegramBotAPI)
- **База данных**: PostgreSQL + SQLAlchemy
- **Планировщик**: APScheduler
- **Парсинг**: Requests
- **Контейнеризация**: Docker, Docker Compose

## 📂 Структура проекта
- **api/** — логика парсинга данных через API Wildberries.
- **config_data/** — загрузка конфигурации и переменных окружения.
- **database/** — описание моделей SQLAlchemy и управление БД.
- **handlers/** — обработчики команд и сообщений (custom и default).
- **keyboards/** — генераторы Inline и Reply клавиатур.
- **utils/** — планировщик задач (scheduler) и вспомогательные скрипты.
- **.env.template** — шаблон переменных окружения для быстрой настройки.
- **docker-compose.yml** — конфигурация для запуска связки бота и базы данных.
- **Dockerfile** — инструкции для сборки образа бота.
- **loader.py** — инициализация объекта бота и логгера.
- **main.py** — точка входа в приложение.

## 🚀 Инструкция по запуску

### 1. Подготовка окружения
Клонируйте репозиторий и перейдите в папку проекта:
```bash
git clone <ссылка_на_ваш_репозиторий>
cd Price_Traker_Bot
```

### 2. Настройка переменных окружения
Создайте файл .env в корне проекта на основе шаблона .env.template. Заполните необходимые данные:
```env
BOT_TOKEN=your_token_from_botfather

DB_NAME=your_db_name
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_HOST=localhost
DB_PORT=5432
```

### 3. Настройка виртуального окружения
```bash
python -m venv venv

# Активация для Windows:
venv\Scripts\activate

# Активация для Linux/macOS:
source venv/bin/activate
```

### 4. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 5. Запуск бота
```bash
python main.py
```

## 🚀 Запуск проекта через Docker

### 1. Подготовка
Убедитесь, что у вас установлен Docker Desktop. Клонируйте репозиторий и перейдите в папку проекта:
```bash
git clone <ссылка_на_твой_репозиторий>
cd Price_Traker_Bot
```

### 2. Настройка переменных окружения
Создайте файл .env в корневой директории.
```env
BOT_TOKEN=your_token_from_botfather

DB_NAME=your_db_name
DB_USER=your_db_user
DB_PASSWORD=your_db_password
# Пример для Docker (используем имя сервиса базы данных)
DB_HOST=db
DB_PORT=5432
```

### 3. Запуск
Выполните команду для сборки образов и запуска контейнеров в фоновом режиме:
```bash
docker-compose up -d --build
```

### 4. Управление
- **Просмотр логов бота**: docker-compose logs -f bot
- **Остановка проекта**: docker-compose down
- **Перезапуск**: docker-compose restart

> **Совет по работе с БД:** Для просмотра таблиц через DBeaver при запущенном Docker используйте `Host: localhost`, `Port: 5432` и данные из вашего `.env`.

## 📝 Дополнительно
- **Persistent Data**: Благодаря Docker Volumes, данные вашей базы сохраняются в папке `postgres_data` даже после остановки контейнеров.
- **Auto-migrations**: База данных инициализируется автоматически при первом запуске (используется SQLAlchemy `create_all`).
- **Logging**: Все критические события и ошибки парсинга пишутся в файл `bot.log` внутри контейнера (доступны через `docker logs`).
